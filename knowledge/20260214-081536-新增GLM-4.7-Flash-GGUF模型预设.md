# 20260214-081536 新增GLM-4.7-Flash-GGUF模型预设

## 目标
在现有启动流程中新增 `GLM-4.7-Flash-GGUF` 量化模型的一键预设能力。

## 本次改动
- `start.sh`
  - 新增默认模型标签：
    - `hf.co/unsloth/GLM-4.7-Flash-GGUF:Q4_K_M`
  - 新增开关：`DEPLOY_GLM47_FLASH_GGUF`
  - 当开关开启且 `OLLAMA_MODEL2` 未设置时，自动注入该模型到 `OLLAMA_MODEL2`
  - 若用户已设置 `OLLAMA_MODEL2`，保持用户配置不覆盖
- `docker-compose.yml`
  - 新增环境变量透传：`DEPLOY_GLM47_FLASH_GGUF`
- `.env.example`
  - 新增开关默认值：`DEPLOY_GLM47_FLASH_GGUF=false`
  - 新增 `OLLAMA_MODEL2` 示例
- `README.md`
  - 环境变量表新增 `DEPLOY_GLM47_FLASH_GGUF`
  - 增加自动注入说明和手动覆盖示例

## 使用方式
启用 GLM-4.7-Flash 预设：

```bash
DEPLOY_GLM47_FLASH_GGUF=true
```

手动覆盖模型标签：

```bash
OLLAMA_MODEL2=hf.co/unsloth/GLM-4.7-Flash-GGUF:Q4_K_M
```

## 说明
该标签为单文件 GGUF，适配当前 Ollama 拉取方式，避免分片 GGUF 导致的拉取失败问题。
