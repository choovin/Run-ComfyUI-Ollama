# 20260214-132054 GLM5侧车默认官方镜像并保留本地构建兜底

## 目标
针对 sidecar 本地编译失败问题，改为优先使用官方预编译镜像，必要时再回退本地源码构建。

## 变更文件
- docker-compose.yml
- .env.example
- README.md

## 关键改动
### 默认路径（推荐）
- llama-cpp-glm5 服务默认改为：
  - LLAMACPP_SIDECAR_IMAGE=ghcr.io/ggml-org/llama.cpp:server-cuda
- 启动命令：
  - docker compose --profile glm5-sidecar up -d

### 兜底路径（本地编译）
- 新增 llama-cpp-glm5-localbuild 服务，profile：
  - glm5-sidecar-localbuild
- 仍使用仓库内 sidecar/llamacpp-glm5/Dockerfile 本地构建。
- 兜底启动命令：
  - docker compose --profile glm5-sidecar-localbuild up -d --build

### 环境变量
- .env.example 新增：
  - LLAMACPP_SIDECAR_IMAGE=ghcr.io/ggml-org/llama.cpp:server-cuda

## 结果
默认路径避免本地 CUDA 链接编译问题；若官方镜像不兼容目标模型，再切换本地构建 profile 进行回退。
