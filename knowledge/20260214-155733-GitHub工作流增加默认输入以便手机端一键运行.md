# 2026-02-14 15:57:33 GitHub工作流增加默认输入以便手机端一键运行

## 背景

用户反馈在手机端手动触发 GitHub Actions 时输入参数不方便，希望 workflow 提供默认选项，减少手动输入。

## 本次修改

1. 更新 `.github/workflows/docker-acr.yml` 的 `workflow_dispatch` 输入：
- `image_version`:
  - 从 `required: true` 调整为可选。
  - 增加默认值 `auto`。
- `sidecar_source_image`:
  - 增加默认值 `ghcr.io/ggml-org/llama.cpp:server-cuda`。

2. 修正版本解析逻辑（两个 job 都同步）：
- 当 `image_version` 为空或为 `auto` 时，自动生成版本：

```bash
YYYYMMDD-<short_sha>
```

- 避免手动触发时出现空标签。

3. 更新 `README.md`：
- 将 `image_version` 说明改为“可选，默认 auto 自动生成版本号”。

## 效果

- 手机端可以直接点击运行，不必手输版本号。
- sidecar 来源镜像也有默认值，通常无需修改。
- 保持标签可追溯性，仍包含日期和提交短 SHA。
