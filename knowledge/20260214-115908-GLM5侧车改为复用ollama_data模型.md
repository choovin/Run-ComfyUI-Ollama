# 20260214-115908 GLM5侧车改为复用ollama_data模型

## 目标
`llama.cpp` sidecar 不再单独挂载 `./models/glm5`，改为直接复用已下载完成的 `ollama_data` 卷中的 GLM-5 模型文件。

## 变更文件
- `docker-compose.yml`
- `.env.example`
- `README.md`

## 具体改动
### docker-compose
- sidecar 服务 `llama-cpp-glm5` 的卷挂载改为：
  - `ollama_data:/root/.ollama:ro`
- sidecar 模型参数改为环境变量路径：
  - `LLAMACPP_GLM5_MODEL_PATH`
- 默认值设置为已下载的 GLM-5 blob：
  - `/root/.ollama/models/blobs/sha256-0c5372e168853083392239349e76fb8a09c60e86150d0d4ebe4b0a0ba5f0882a`

### .env.example
- 删除 `LLAMACPP_GLM5_MODEL_FILE`
- 新增 `LLAMACPP_GLM5_MODEL_PATH` 并给出默认 blob 路径

### README
- sidecar 使用说明改为“复用 Ollama volume 模型”
- 同步替换环境变量示例为 `LLAMACPP_GLM5_MODEL_PATH`

## 结果
无需重复下载 GLM-5 模型文件，sidecar 可直接使用主容器 `ollama_data` 中已存在的模型。
