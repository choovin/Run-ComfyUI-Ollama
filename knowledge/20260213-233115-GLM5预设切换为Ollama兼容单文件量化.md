# 20260213-233115 GLM5预设切换为Ollama兼容单文件量化

## 背景
运行日志报错：
- `The specified tag is a sharded GGUF. Ollama does not support this yet.`

原因为 `hf.co/unsloth/GLM-5-GGUF:UD-IQ2_XXS` 为分片 GGUF，当前 Ollama 不支持直接拉取分片标签。

## 变更
- `start.sh`
  - 保留开关 `DEPLOY_GLM5_UD_IQ2_XXS`（兼容旧配置）
  - 默认注入模型改为 Ollama 可加载的单文件标签：
    - `hf.co/unsloth/GLM-5-GGUF:UD-TQ1_0`
  - 增加日志说明“使用 Ollama-compatible tag”
- `.env.example`
  - `OLLAMA_MODEL1` 示例由 `UD-IQ2_XXS` 改为 `UD-TQ1_0`
- `README.md`
  - 自动注入示例与手动覆盖示例同步改为 `UD-TQ1_0`
  - 补充原因说明：`UD-IQ2_XXS` 为分片 GGUF，当前 Ollama 不支持

## 结果
启用 `DEPLOY_GLM5_UD_IQ2_XXS=true` 时，项目会自动选择当前 Ollama 可拉取的 GLM-5 标签，避免启动阶段 `ollama pull` 失败。
